## 1. Wildberries

### Что было не так
1. **Слой цен (`sales`) жил по старой модели периодов.** Витрина `margin_by_sku` — по месяцу
   формирования отчёта, а `sales` (наша цена, цена ВБ, СПП) писался по окну запроса.
   Дашборд джойнил их по периоду → «наша цена и цена ВБ не сходятся».
2. **СПП в агрегате главной** считался старой формулой → показывал 0.1%.
3. **Нулевые отгрузки МС проходили как «покрытые»:** отгрузка есть в кэше, но себест = 0
   (МС не знает цену) — матч засчитывался, ноль ложился в COGS, чистая по этим строкам завышалась.

### Что сделано
- Новый `reports/wb_sales_formation.py`: sales пересобирается из сырья по месяцу формирования —
  та же модель, что у витрины. Старая запись sales в коллекторе отключена (один писатель).
  Встроен в run_daily после витрины.
- Витрина: нулевой себест в кэше = НЕсматченный → уходит в цепочку фолбэков
  (средняя месяца → история → группа-префикс → набор → закупочная).
- СПП = (наша цена − «ВБ реализовал») / наша цена. В агрегат добавлены revenue_wb / margin_wb.

### Проверки
- Витрина = sales по выручке копейка-в-копейку (июнь: 4 179 585 = 4 179 585).
- СПП по месяцам 25.2–28.5% — согласуется с ЛК (~26–29%).
- Строк с qty>0 и COGS=0: **0** (было 58 строк / 68 шт).
- Контроль nmID 216421567 май: чистая 3908 ₽ (эталон ≈3909).

## 2. ФБО и себестоимость ВБ

- Модель (с 12.07): COGS = себест отгрузок МС, матч по assembly_id напрямую; FBO (~11%) —
  импутация из FBS-отгрузок того же артикула + наборы через mix_data. Покрытие **100%**
  во всех 16 месяцах×аккаунтах — теперь без нулевых дыр.
- **186 отгрузок МС с нулевым себестом** (55 Цифровой, 131 Дисквэр) — список для правки:
  `docs/ms_zero_cogs_demands.csv` (+ скачивание: https://bi.metaverseworld.ru/download/ms-zero-cogs.csv).
  Кэш по ним сброшен и перечитан 13.07 — в МС пока не исправлено (183 из 184 всё ещё 0),
  их штуки временно закрыты фолбэком (средняя по артикулу). После правки в МС:
  `DELETE FROM ms_demand_cogs WHERE cogs=0` → пересбор кэша → пересборка витрины.

## 3. Яндекс.Маркет

### Было
Заказы только за ~30 дней; расходы одним числом без месяцев; блок на дашборде не реагировал
на выбранный месяц; возвраты УДВАИВАЛИ выручку (REFUND приходит с плюсом).

### Стало
- История с января: 2375 заказов (raw_yandex_stats_order), каталог 15 156 офферов (raw_yandex_offer).
- `yandex_finance_monthly`: выручка (возвраты вычтены), субсидия, **комиссия, логистика
  (вкл. межрегиональную и экспресс), эквайринг, буст-реклама, агентское/сортировка/лояльность**,
  возвраты, COGS.
- **COGS = факт из МС-заказов Маркета** («Покупатель Маркет» + «Я.Маркет Экспресс») по месяцам,
  покрытие 100%. Сверка: Σ янв–июл 2.62 млн ≈ 2.66 млн (МС за 200 дней). Связка по баркоду
  не взлетела (баркоды ЯМ сгенерированные, матч 17%), закупочная из карточек местами = розница —
  карта по SKU оставлена только фолбэком.
- Дашборд: блок Маркета на главной — за выбранный месяц, входит в ИТОГ и дельты;
  на /market — селектор месяца + таблица экономики по месяцам.
