# WB: где проигрываем по логистике (габариты карточек vs реальные)

Дата: 2026-07-15. Домен: fin. Источник данных: `supplier_dims` (габариты закупочной упаковки
4 поставщиков) × `wb_cards.volume_l` (объём, задекларированный в карточке WB) × `margin_by_sku`
(факт логистики и продаж, июнь 2026).

## Суть

WB считает логистику FBS по **объёму задекларированного короба**. Тариф, выведенный из наших же
данных (регрессия факт-логистики на объём карточки):

```
логистика/шт ≈ 46 + 29,8 × объём_л
1 л → 76 ₽   5 л → 195 ₽   11 л → 374 ₽
```

Каждый лишний литр в карточке = **~30 ₽ переплаты за штуку**. А карточки раздуты: медиана
отношения `объём_карточки / реальный_объём_поставщика` = **2,30×**. 148 карточек висят на
дефолтных ~11 л, тогда как реальный медианный объём картриджа — **4,58 л**.

## Потенциал (по покрытой части ассортимента)

- Матчинг nm↔артикул поставщика: **406 nm** из 1342 продающих за июнь (~30%; ограничено тем, что
  это лишь 4 поставщика из N — «это не все»).
- WB-логистика июнь всего: **893 595 ₽** на 3004 шт (сред. 297 ₽/шт).
- **Потенциал экономии при коррекции карточек до реального объёма: ~133 000 ₽/мес** (≈15% всей
  логистики WB), из них 116к — на 237 карточках, раздутых более чем вдвое.
- Экстраполяция на весь ассортимент (если картина типична) — до ~2× выше.

## Как действовать

1. Взять топ CSV, у каждого nm сверить реальные габариты картриджа с задекларированными в карточке WB.
2. Где карточка раздута — исправить ДхШхВ в карточке (Контент WB) на реальные. Пересчёт тарифа у WB
   произойдёт автоматически со следующих поставок.
3. Начать с 148 «дефолтных ~11 л» — это явно незаполненные габариты, а не осознанный короб.

## Оговорки (почему это оценка сверху)

- Габариты поставщика — это **закупочная упаковка одного картриджа**, прокси реального размера. Наш
  отгрузочный короб может быть больше (несколько штук, наполнитель) → фактическая экономия ниже.
- Покрыты только 4 поставщика (~30% продаж). Остальное появится по мере добавления прайсов.
- Тариф выведен эмпирически из наших данных, не из офиц. сетки WB — годится для приоритизации, не для
  точного биллинга.
