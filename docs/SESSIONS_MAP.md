# SESSIONS_MAP — карта рабочих сессий (tmux × worktree × поток)

> Зачем: сессии копятся (mkt→mkt2→mkt3→mkt4, reviews/review3…), имена версионируются вручную,
> старые дубли висят с прогоревшим контекстом. Эта карта — единый реестр: **один поток = одна живая
> tmux-сессия с каноничным именем**. Обновлять при заведении/сворачивании сессии. Обновлено 2026-07-21.

## Правила именования и жизненного цикла

1. **Одно каноничное имя на поток** — ровно из таблицы выше, без версий (`mkt`, а не `mkt3/mkt4`).
2. **Версионные суффиксы `-N` запрещены на живых сессиях.** Наследование при переполнении контекста идёт
   через `docs/HANDOFF.md` (+ доменный `HANDOFF_*.md`), а НЕ через новую сессию `xxxN`. Переполнилась —
   передаёшь состояние в HANDOFF, `/clear` или новый процесс под ТЕМ ЖЕ именем; предшественника **убить**.
3. **Предшественника убивают, не переименовывают.** После переноса в HANDOFF старую tmux-сессию сносят
   (`tmux kill-session -t <old>`), чтобы не висел прогоревший дубль (mkt3/mp-mkt2 = анти-пример).
4. **Один поток — свой worktree/ветка.** Несколько сессий на одном чекауте `/opt/mp-analytics` (сейчас так
   у reviews/dims) — гонка правок; каждому потоку завести свой worktree, как у mkt/fin.
5. **Голая `claude`-сессия — не заводить.** Имя должно называть поток. Сгруппированные сессии
   (`new-session -t`), показывающие один и тот же процесс под вторым именем, — убить лишнее имя.
6. **Чужие проекты — по своей директории** (`china-audit` в `/opt/china-audit`). В этот реестр входят
   ссылкой, но правила mp-analytics на них не распространяются.

### Прямые ответы на вопросы уборки
- **mkt3 / mkt4 / mp-mkt2 — актуальна `mkt4`** (= текущая живая сессия). mkt3 и mp-mkt2 — прогоревшие
  предшественники по цепочке HANDOFF, снести.
- **reviews / review3 — актуальна `review3`.** `reviews` — мёртвый bash-шелл, снести.
- **claude vs mp-night — не про финансы.** `claude` = лишнее имя того же процесса, что `mkt4` (снести).
  `mp-night` финансы НЕ дублирует — она в потоке отзывов; пересекается с `review3`, не с финансами.
  Отдельной живой финансовой сессии среди этих 11 вообще нет.
