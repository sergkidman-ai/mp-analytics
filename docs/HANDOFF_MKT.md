## 1. Падение видимости/трафика — где точка и остановилось ли

Месячный ряд (скользящее 7-дн окно, метрика «входы в карточку» = трафик из поиска):

| Окно | Видимость | avg-поз | Входы (трафик) |
|---|---|---|---|
| 20–26 май | 79 | 12 | 3020 |
| 28 май–03 июн | **89** | 11 | **3510 ← пик** |
| 05–11 июн | 79 | 11 | 3014 |
| 11–17 июн | 79 | 12 | 2788 |
| 17–23 июн | 79 | 15 | 2360 |
| **19–25 июн** | **71** ↓ | **20** ↓ | 2151 ← дно |
| 23–29 июн | 71 | 16 | **2231** ↑ |

- **Точка падения — начало июня** (сразу после пика 28 мая–03 июня). Трафик сполз **3510 → ~2150 (−37%)** постепенно.
- **Видимость** жёстко просела **79→71 в окне с 19 июня** (avg-поз 11→20).
- **Последняя неделя — первые признаки дна:** трафик 2151→2231 (вверх), avg-поз 20→16.
- Новая формула акций с **26 июня** (пятница) попадает в этот разворот, но в 7-дн метрике пока хвост
  одного окна — доказать эффект акций рано, нужно 3–4 дня свежих окон.
- Скрипт месячного ряда был разовый (`scratch_jam_month.py`, удалён) — логика: report-endpoint с
  произвольным `currentPeriod`, `limit=1`, читаем `visibilityInfo`/`positionInfo`. Данные сохраняются
  в `wb_search_summary` (UPSERT по period_start). Можно оформить в коллектор для авто-замера.

## 2. Воронка топ-30 (30-дн окно 31.05–29.06) + точки потери

Медиана по топу: **вход→корзина 16%, корзина→заказ 42%**. Три типа потерь:
- 🔴 **корзина→заказ низкая** (069H Canon 8%, Canon Selphy 23%, Lexmark/Samsung ~20%) — кладут в
  корзину, покупают НЕ у нас → цена/отзывы/доставка = проигрыш конкуренту на финише.
- 🔴 **вход→корзина низкая** (Чернила №110 15%, 067H 11%, TL-420Е 7%) — карточка/цена/превью/рейтинг.
- ✅ здоровые: HP 1018 (вход162→корз67→зак29), HP 1020, Samsung ML-2160/M2020 — лить больше трафика.

**Главная точка роста — `Canon Selphy фото`:** топ трафика **213 входов**, поз 4, видимость 100%, но до
заказа доходит **6**. Чинить корзину→заказ (цена/отзывы) = сразу деньги, спрос уже есть.

Разовый скрипт воронки удалён (`scratch_funnel.py`); логика: report-endpoint `orderBy openCard desc
limit 30`, поля openCard/addToCart/orders/openToCart/cartToOrder. Стоит оформить витриной `reports/funnel*`.

## 4. Видим ли конкурентов и их ключи — НЕТ (важно)
API «Джем» (`search-report/report`, `product/search-texts`) отдаёт **только наши товары**. Конкретных
карточек конкурентов, их цен и ключей — нет. Косвенно конкуренцию видно: наша позиция по запросу
(сколько чужих впереди), видимость <100% (остаток забрали конкуренты), и 🔴 корзина→заказ = прямой
след «купили дешевле у другого». Реальная конкурент-разведка = WB-аналитика категорий / внешние
сервисы (MPSTATS и т.п.) / вручную.

## 5. Потенциал данных (что строить, всё уже собирается)
1. **Воронка × маржа** = приоритет фиксов в рублях (маржу брать READ-ONLY из `margin_by_sku`). Кандидат №1 — Canon Selphy.
2. Деньги за позицию/запрос (заказы из поиска × маржа).
3. ROI рекламы по товару (`is_advertised` + `wb_ads`/`ad_spend_daily` + заказы + маржа).
4. Спрос-предложение gap (ВЧ-запросы, где мы невидимы, но есть остаток+маржа).
5. Цена vs конверсия (падение корзина→заказ ↔ наша цена).
6. Авто-детектор просадок → таблица `drops` (уже есть), причина из этапа воронки.
