# HANDOFF — состояние проекта по потокам

> **Читать ПЕРВЫМ делом при старте сессии** — свою секцию потока (ниже). Обновлять в конце значимого блока.
> Карта потоков и сессий — `docs/SESSIONS_MAP.md`; правила территорий — `CLAUDE.md`. Обновлено: 2026-07-21.
> Каждая сессия ведёт СВОЮ секцию; чужие секции не трогать (нужна чужая территория → пиши вопрос в свою секцию).
> worktree-сессии при старте подтягивают доки из `main` (`git show main:CLAUDE.md` / `git show main:docs/HANDOFF.md`) — веточная копия бывает stale.

## Вход в сессию (шпаргалка)
```
ssh <сервер>
tmux ls                 # список сессий; каноничные имена: fin mkt rev gab inv china
tmux attach -t <имя>    # напр. tmux attach -t mkt
# внутри tmux:  Ctrl+b d — отсоединиться (сессия живёт);  tmux switch-client -t <имя> — перейти в другую
```

## Процедура переезда сессии
Переезд = чистый рестарт Claude в той же tmux-сессии (напр. при переполнении контекста). **Tmux-сессии
при переезде НЕ пересоздаются** — новый `claude` стартует в уже существующем окне того же потока.
1. **Дежурному (перед выходом):** обнови свою секцию потока в этом `HANDOFF.md` (состояние / сделанное /
   активные ветки / открытые вопросы / следующий шаг) и **закоммить всё незакоммиченное** в своей ветке.
2. `/exit` — завершить текущую сессию Claude.
3. `claude` — запустить заново в ТОЙ ЖЕ tmux-сессии (окно потока не трогаем).
4. Стартовое сообщение новой сессии:
   > «Ты сессия потока **X**, прочитай `CLAUDE.md` + свою секцию `HANDOFF` + `BRIEF`, доложи и продолжай».

- **Политика периода (согласовано словами):** отчётный месяц определяет **дата формирования** отчёта ВБ —
  отчёт падает в месяц целиком (выручка+расходы+себест), без дробления по `rr_dt`; правило для ВСЕХ отчётов
  и организаций. Контроль клиента: «за июнь **8 отчётов**» (была ошибка разбивки). «**Итого к оплате** брать
  СТРОГО из отчёта ВБ, не рассчитывать».
- **Якоря сверки «К перечислению» ВБ, июнь по неделям (ЛК):** 01.06=604 386, 08.06=621 963, 15.06=487 337,
  22.06=423 689, 29.06=392 734 ₽ — эталон при проверке недельной таблицы.
- **COGS byoperation vs эталон:** переход `cost_seb`→`byoperation` дал Янв-Апр итог **−9.0%**, после фикса
  модели периода сошлось до **−0.9…−1.8%**. Контрольная отгрузка `4747758355` = **284.00 ₽** ровно.
- **FBO-импутация — правило префикса (клиент):** у 5-значного артикула себест берётся по **первым 4 знакам**
  (= товар); закрыло 34/49 «мёртвых». Мёртвый хвост FBO: **59 шт за 7 мес (дек–июн), все по 1 шт, ~158 тыс ₽**;
  отгрузки окт–ноя «уйдут со временем» (появятся в кэше). Покрытие FBO прямым матчем низкое (~29%), добор импутацией.
- **Себест — единственный источник (зафиксировано клиентом, H380):** `report/stock/byoperation`, «работаем
  только с ней»; НЕ `cost_seb`/`buyPrice`/усреднение.
- **Яндекс — открытые проверки:** «15 возвратов/мес — мало, должны быть **незаборы**» (проверить полноту
  возвратов); реклама ЯМ = **буст показов + буст продаж (% от продаж) + Полки + программы стимуляции** (часть
  Яндекс мог закрыть) — убедиться, что учтены все, не только продвижение.
- **Промо-гипотеза (граница с mkt, H1237):** маржа/видимость решается «**проходом в акции**» — там раздаётся
  СПП, даёт запас цены вниз + привлекательность; «просто реклама не спасёт», ставка на подтяжку длинного хвоста
  к ВЧ-фразам. Юнит-экономика: `чистая/шт = наша_цена×K − логистика − себест`, себест из `margin_by_sku` (с BOM
  наборов). Разобрать перед экспериментом со ставками.
