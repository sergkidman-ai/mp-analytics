## КЛЮЧЕВОЙ РАЗВОРОТ (проверено эмпирически, подтверждено клиентом)

Мёртвая сессия a11ce059 заключила: «себест отгрузки = `report/stock` на ДАТУ по позициям»
(дало 275.99 на эталон-отгрузке 4747758355 при «МС 284»). **Это оказалось приближением, а не
фактом.** Проб на той же отгрузке:

| Источник | Σ себест отгрузки 4747758355 |
|---|---|
| `report/stock/all?moment=11.03` (stock-на-дату) | **275.99** (приближение, ремейн-сток avg) |
| `report/profit/byproduct` усреднён за март | 285.00 |
| **`report/stock/byoperation?operation.id=<demand>`** | **284.00** ✓ = эталон «МС 284» |

**ВЕРНЫЙ ИСТОЧНИК (клиент 2026-06-30: «в отгрузке себест уже рассчитана, возьми ровно её»):**
`GET /report/stock/byoperation?operation.id=<demand_id>` — «Остатки по документу». Возвращает
`rows[0].positions[].cost` (копейки, ИТОГ с учётом quantity) — себест FIFO **на `moment` документа
отгрузки** (для Отгрузки/Розн.продажи/Приёмки/Возвратов считается на moment док-та; док.
МойСклад подтверждает). Σ positions.cost/100 = себест всей отгрузки = ровно то, что МС показывает
в документе. Эталон 4747758355: T036 12600коп + T037 15800коп = 28400коп = **284.00 ₽** ✓.

- `report/profit/byoperation` — НЕ существует (404 «Неопознанный путь»). Не путать с byproduct.
- Дубль-позиции в byoperation агрегируются — нам ОК, т.к. суммируем cost по документу.
