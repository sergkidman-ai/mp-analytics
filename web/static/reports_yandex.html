Яндекс Маркет — сводный отчёт по месяцам

Как читать этот отчёт

Данные из витрины Яндекс.Маркета (Партнёр-API: заказы + отчёт услуг + закрытие месяца) + себестоимость из МойСклад. Оборот = «Оплата покупателя» + «Субсидия Маркета» — от него считаются все % и подсветка (субсидия у ЯМ значимая, до ~64% сверх оплаты). Строки — водопад: оборот → минус комиссия, логистика, перевод, продвижение, агентское, прочие, подписка, баллы → Итого к перечислению → минус COGS → Чистая. Одно юрлицо (Цифровой квадрат) — одна таблица. Столбцы — календарные месяцы. Справа — доля от оборота и тренд. Подсветка — три блока относительно среднего: зелёное = выше среднего (хорошо), янтарное = ниже среднего (обратить внимание), без заливки — норма. Для расходов инверсия (ниже — лучше). Два правых столбца — текущий месяц (оценка) и прогноз на конец месяца, живьём из БД.

выше среднего — хорошо ниже среднего — обратить внимание в норме (около среднего) наша не из отчёта МП (МойСклад) расчёт производная над константами

Все строки — из витрины Яндекс.Маркета (Партнёр-API). Оборот = Оплата покупателя (payment) + Субсидия Маркета (subsidy) — субсидия у ЯМ идёт сверх оплаты и значима. «Итого расходы Маркета» = комиссия + логистика + перевод/эквайринг + продвижение (буст-продажи + буст-показы + полки) + агентское + прочие + подписка + баллы за отзывы. «Итого к перечислению» = Оборот − Итого расходы Маркета. Себестоимость — order-based из МойСклад (Σ себест×кол-во по позициям заказа в месяц заказа; сторно возвратов в месяц заказа, кроме склада «Брак»). ⚠ Живой месяц Яндекса структурно занижает маржу: COGS списывается в месяц заказа целиком, а выручка/субсидия реализуются по мере доставки — часть заказов ещё «в пути» (несут себест до реализации выручки). Поэтому «тек.» — заниженный факт «пока набрано», а «прогноз» проецирует полный месяц по юнит-экономике закрытых месяцев × ожидаемое число заказов (маржа-прогноз сходится к норме). Данные Ozon и WB — на соседних вкладках.
